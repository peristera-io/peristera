# Scaleway deploy (M7)

OpenTofu + a bootstrap script that stand up the Peristera public deployment on
Scaleway: a single instance running k3s behind a reserved Flexible IP, with the
full platform on real Let's Encrypt TLS. The architecture and its trade-offs
are ADR-0020; this README is the operator runbook.

## Topology (why k3s, not Kapsule)

A single instance + k3s + a reserved Flexible IP gives a **stable ingress IP
with no managed Load Balancer** and mirrors dev (k3d→k3s), so the same
manifests and CNI (Cilium) carry over. Kapsule + a managed LB (a ~€50/mo floor)
is deferred to real multi-node scale. Node `PLAY2-MICRO` (8 GB) ≈ €0.055/hr
(~€40/mo if left running); the cost model is **on-demand** — stop the instance
to pause at ~€2/mo with state intact, `tofu destroy` to zero.

## Layout

```text
*.tf                     infra: instance, Flexible IP, firewall, Object Storage,
                         Secret Manager (generated platform secrets)
backend.tf.example       remote-state (Object Storage) — opt-in, see below
manifests/               cloud platform manifests (envsubst ${DOMAIN} etc.)
bootstrap.sh             cloud twin of hack/dev-cluster.sh — brings up the platform
```

## Guardrails

- **Secrets/state never touch git.** SCW creds come from `../../.env.scaleway`
  (gitignored). Platform secrets are generated by Tofu straight into Scaleway
  Secret Manager and synced in by ESO. State can hold secret values, so real
  deploys keep it in the Object Storage backend, not on disk.
- **Billing:** always `tofu plan` before `apply` (review the resource list),
  `tofu destroy` (or stop the instance) when done. A budget alert is the
  backstop. The deploy key has no billing permissions.
- **Firewall:** set `TF_VAR_admin_cidr` to your `/32` before apply — it gates
  SSH and the k3s API (6443). The default is an unusable placeholder that fails
  closed (Scaleway opens 6443 to the internet by default).

## Prerequisites

- `peristera.app` delegated to **Scaleway DNS** (external-dns writes records).
- Images pushed to `ghcr.io/peristera-io/<app>:<tag>` (the `images.yml` CI job).
- Tools: `tofu`, `kubectl`, `helm`, `envsubst`, `curl`, `python3`.

## Deploy

```sh
set -a && . ../../.env.scaleway && set +a          # SCW_* creds
export TF_VAR_admin_cidr="$(curl -s ifconfig.me)/32"
export TF_VAR_letsencrypt_email="you@example.org"

tofu init
tofu plan                                          # review resources + cost
tofu apply                                         # provisions (spends money)

# fetch the node kubeconfig (see the kubeconfig_hint output)
eval "$(tofu output -raw kubeconfig_hint)"
export KUBECONFIG=/tmp/peristera.kubeconfig

# bring up the platform on real https
export DOMAIN="$(tofu output -raw domain)"
export LE_EMAIL="$TF_VAR_letsencrypt_email"
export IMAGE_TAG=0.0.1-alpha
./bootstrap.sh
# -> https://cp.<domain>  and  https://iam.<domain>  on real certs
```

## Remote state (real deploys)

Off by default so a fresh `tofu init` uses local state. To move state to Object
Storage (two-phase, so `destroy` never deletes its own store):

```sh
scw object bucket create name=peristera-tfstate region=fr-par   # once, out of band
export AWS_ACCESS_KEY_ID="$SCW_ACCESS_KEY" AWS_SECRET_ACCESS_KEY="$SCW_SECRET_KEY"
cp backend.tf.example backend.tf
tofu init -migrate-state
```

## Pause / resume / teardown

```sh
# pause (keeps volume + Flexible IP, stops the meter for compute):
scw instance server stop "$(tofu output -raw node_ip | xargs -I{} echo {})"   # or via console
# resume: start the server, then re-run ./bootstrap.sh (idempotent)
# full teardown:
tofu destroy
```

## Scope

s1 stands up the **platform** (control plane + Zitadel) on TLS. Tenant
provisioning on real per-host TLS — each tenant's Zitadel-issuer and app
ingresses getting their own certs — is **s2** (control-plane work), building on
the `TENANT_SCHEME=https` knob wired here.
